Introducción

SQL, Structure Query Language (Lenguaje de Consulta Estructurado) es un lenguaje de programación para trabajar con bases de datos relacionales como MySQL, Oracle, etc.
MySQL es un interpretador de SQL, es un servidor de base de datos.
Con SQL podemos crear bases de datos y tablas, insertar datos, hacer consultas y realizar muchas operaciones, en resumen: administrar bases de datos.
MySQL es un sistema en línea de comandos o también, en lenguaje común, PHP nos permite trabajar con bases de datos desde aplicaciones web.
La sensibilidad a mayúsculas y minúsculas depende del sistema operativo. En Linux sí hay diferencia, mientras que en Windows no.
Ejemplo: Create database administracion;
Crear base de datos administración.
Se recomienda usar siempre minúsculas para la segunda.

show databases

Una base de datos es un conjunto de tablas.
Vamos a trabajar con una base de datos y crearla en el sitio, llenando “administracion”.
Podemos obtener los nombres de las bases de datos existentes con el comando:
show databases;
Nos mostrará los nombres de las bases de datos, debe aparecer “administracion”.

Creación de una tabla y mostrar sus campos

Una base de datos almacena sus datos en tablas.
Una tabla es una estructura de datos que organiza los datos en columnas y filas; cada columna es un campo (o atributo) y cada fila, un registro. La intersección de una columna con una fila, contiene un dato específico, un solo valor.
Cada registro contiene un dato por cada columna de la tabla.
Cada campo (columna) debe tener un nombre. El nombre del campo hace referencia a la información que almacenará.
Cada campo (columna) también debe definir el tipo de dato que almacenará.

Carga de registros a una tabla y su recuperación

Usamos "insert into". Especificamos los nombres de los campos entre paréntesis y separados por comas y luego los valores para cada campo, también entre paréntesis y separados por comas.
Es importante ingresar los valores en el mismo orden en que se nombran los campos, si ingresamos los datos en otro orden, no aparece un mensaje de error y los datos se guardan de modo incorrecto.
Note que los datos ingresados, como correspondan a campos de cadenas de caracteres se colocan entre comillas simples. Las comillas simples son OBLIGATORIAS.
