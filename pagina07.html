Introducción


SQL, Structure Query Language (Lenguaje de Consulta Estructurado) es un lenguaje de programación para trabajar con base de datos relacionales como MySQL, Oracle, etc.
MySQL es un interpretador de SQL, es un servidor de base de datos.
Con SQL podemos crear bases de datos y tablas, insertar datos, hacer consultas y realizar muchas operaciones, el resumen: administrar bases de datos.
MySQL es un sistema en línea de comandos o también en lenguaje común PHP nos permite trabajar con bases de datos desde aplicaciones web.
La sensibilidad a mayúsculas y minúsculas, es decir, hace diferencia entre ellas, dependiendo del sistema operativo, especialmente para Linux sí. Por ejemplo Windows no hace diferencia.

Mostrar bases de datos


Una base de datos es un conjunto de tablas.
Vamos a trabajar con una base de datos y crearla en el sitio, llenando “administracion”.
Podemos obtener los nombres de las bases de datos existentes o solicitarlas enviando un mensaje.
show databases;
Nos mostrará los nombres de las bases de datos, debe aparecer en este sitio “administracion”.

Creación de una tabla y mostrar sus campos


Una base de datos está compuesta por una o varias tablas.
Cada tabla está compuesta por columnas y filas; cada columna representa un campo, fila un registro. La intersección de una columna y una fila representa un dato, un solo valor.
Cada campo debe tener un nombre dentro de la tabla.
El nombre del campo debe ser único. El nombre del campo hace referencia a la información que almacenará.
Cada campo (columna) también debe definir el tipo de dato que almacenará.

Carga de registros a una tabla y su recuperación

Usamos "insert into". Especificamos los nombres de los campos entre paréntesis y separados por comas y luego los valores para cada campo, también entre paréntesis y separados por comas.
Es importante ingresar los valores en el mismo orden en que se nombran los campos, si ingresamos los datos en otro orden, no aparece un mensaje de error y los datos se guardan de modo incorrecto.
Note que los datos ingresados, como corresponden a campos de cadenas de caracteres se colocan entre comillas simples. Las comillas simples son OBLIGATORIAS.
